Unclosed Tags

What it means

HTML element is missing a closing tag, which can cause rendering issues or unexpected behavior on the page.

Why it matters

Unclosed tags can break the structure of a webpage, causing layout issues and preventing elements from displaying correctly.

How to fix it

Ensure that all HTML tags are properly closed with a matching closing tag.

How we determine

We scan the HTML document for tags without a corresponding closing tag and flag them for review.

<a> ref has missing link

What it means

An anchor (<a>) tag is missing a valid href attribute, making the link non-functional.

Why it matters

Links without valid hrefs are broken, and users may not be able to navigate or access important content.

How to fix it

Ensure all anchor tags have a valid href attribute pointing to the desired destination.

How we determine

We check all <a> elements for the presence of a valid href attribute.

Missing Headers

What it means

Important content sections are not properly marked with header tags (<h1>, <h2>, etc.), making it harder for users and search engines to understand the structure of the page.

Why it matters

Missing headers can affect the accessibility of your page, as screen readers rely on them to navigate content. It also affects SEO rankings.

How to fix it

Ensure every important section of content is preceded by a proper header tag.

How we determine

We analyze the structure of the page to identify sections without appropriate headers.

Missing Input Label

What it means

An input element (<input>) is missing a label, which helps screen readers and users understand the purpose of the input field.

Why it matters

Without labels, users with visual impairments or those relying on screen readers may have difficulty using the form fields.

How to fix it

Add a <label> element for every form input, using the for attribute to associate the label with the input field.

How we determine

We check if each form input has a corresponding <label> element linked via the for attribute.

Bad Contrast (Hard to See)

What it means

The contrast between text and its background is too low, making it difficult for users with visual impairments to read the content.

Why it matters

Low contrast can make content inaccessible to users with visual disabilities or those viewing in poor lighting conditions.

How to fix it

Ensure sufficient contrast between text and background colors, aiming for at least a 4.5:1 ratio for regular text and 3:1 for large text.

How we determine

We analyze color combinations to check if they meet the WCAG contrast ratio standards.

Small Text

What it means

Text on the page is too small, making it hard to read for many users, especially those with visual impairments.

Why it matters

Small text can make it difficult for users to read important content, especially on mobile devices or for users with vision problems.

How to fix it

Increase the font size of body text to a minimum of 16px, and use relative units like em for scalability.

How we determine

We check the font sizes used throughout the page to ensure they meet accessibility standards.

Skipped Header Level

What it means

Headers on the page skip a level, making the page structure confusing for screen readers and disrupting the logical flow of content.

Why it matters

Skipped header levels can disrupt content navigation, making it harder for users to understand the hierarchy and structure of the page.

How to fix it

Ensure headers follow a logical order, starting from <h1> down to <h6> without skipping levels.

How we determine

We check the sequence of header elements to ensure they follow a proper hierarchy.

Missing Lang Attribute

What it means

The <html> tag is missing a lang attribute, which helps screen readers and search engines identify the language of the document.

Why it matters

Without a lang attribute, screen readers may mispronounce words, and search engines may have difficulty indexing content properly.

How to fix it

Add the lang attribute to the <html> tag with the correct language code, e.g., <html lang="en"> for English.

How we determine

We check the <html> element to ensure it has a valid lang attribute.

Redundant Link

What it means

A link is pointing to the same resource as the current page or leads to an unnecessary redirection.

Why it matters

Redundant links waste resources and confuse users, as they do not provide any new or useful content.

How to fix it

Remove any redundant links or ensure they lead to distinct resources.

How we determine

We analyze the links on the page and check if any point to the same URL or lead to unnecessary redirects.
